# Social Media Demo — AI Employee Vault Gold Tier

This guide shows how to run the end-to-end social media publishing demo in
under 60 seconds using the MCP `publish` tool and the HITL approval workflow.

## How to Run the Demo in 60 Seconds

### Step 1 — Drop a demo task into Inbox (5 sec)

Copy any of the pre-built scenarios into the agent's inbox:

```bash
# Facebook post demo
cp Demo_Scenarios/facebook_post_demo.md Inbox/

# Instagram post demo
cp Demo_Scenarios/instagram_post_demo.md Inbox/

# Twitter / X post demo
cp Demo_Scenarios/twitter_post_demo.md Inbox/
```

### Step 2 — Start Gold Agent (5 sec)

```bash
python gold_agent.py
```

The agent will:
- Pick up the file from `Inbox/`
- Detect sensitive keywords (`publish`, `post to`, `broadcast`)
- Route it to `Pending_Approval/` and write an HITL approval request

### Step 3 — Approve the action (10 sec)

```bash
# List pending approvals
ls Pending_Approval/

# Approve a specific task
python approve.py hitl_<timestamp>_facebook_post_demo.md

# Or approve everything at once
python approve.py --all
```

### Step 4 — Watch the MCP tool fire (5 sec)

After approval, the agent resumes and calls `browser_mcp_stub.handle_publish()`.

In **DRY_RUN=true** (default) mode you will see:

```
[browser_mcp_stub] DRY RUN: Content publication (post / broadcast / announce)
[browser_mcp_stub]   action_type: publish
[browser_mcp_stub]   task_file:   facebook_post_demo.md
[browser_mcp_stub]   domain:      business
[browser_mcp_stub]   -> Set env MCP_DRY_RUN=false to enable real execution.
```

### Step 5 — Confirm audit log (5 sec)

```bash
ls Logs/ | grep publish
# browser_mcp_stub_publish_dry_run_<timestamp>.json

cat Logs/browser_mcp_stub_publish_dry_run_<timestamp>.json
```

**Total time: ~30 seconds.**

## To Run with Real Publishing (DRY_RUN=false)

> **Warning:** This sends a real post to the social platform. Ensure credentials
> are configured before enabling.

```bash
# Set environment variable
export MCP_DRY_RUN=false

# Add platform credentials (only needed for real execution)
export FB_PAGE_ACCESS_TOKEN="your-token"    # Facebook
export IG_ACCESS_TOKEN="your-token"         # Instagram
export TWITTER_BEARER_TOKEN="your-token"    # Twitter / X

# Then run as normal
python gold_agent.py
```

Replace the `TODO` block inside `mcp/browser_mcp_stub.py` under the `publish`
action type with your real API calls (see inline comments for examples).

## Where Audit Logs Appear

| Location | Format | Contents |
|----------|--------|----------|
| `Logs/` | One JSON file per action | Server name, action type, task file, dry_run flag, timestamp, intent payload |
| `Pending_Approval/` | Markdown + YAML frontmatter | HITL approval requests awaiting human review |
| `Done/` | Processed markdown | Completed tasks with AI summary |
| `Briefings/` | Markdown | Weekly CEO briefing auto-generated by `ceo_briefing.py` |

**Example log filename:**
```
Logs/browser_mcp_stub_publish_dry_run_2026-02-25T09-00-00Z.json
```

**Example log contents:**
```json
{
  "server": "browser_mcp_stub",
  "action": "publish_dry_run",
  "timestamp": "2026-02-25T09:00:00Z",
  "data": {
    "action_type": "publish",
    "task_file": "facebook_post_demo.md",
    "domain": "business",
    "dry_run": true,
    "note": "Set MCP_DRY_RUN=false to enable real actions."
  },
  "success": true
}
```
